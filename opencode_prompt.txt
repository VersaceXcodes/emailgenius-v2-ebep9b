## Backend Connection (CRITICAL)

### ⚠️ CRITICAL CONSTRAINTS - DO NOT VIOLATE
- **DO NOT** remove the environment variable check from API URL configuration
- **DO NOT** replace `import.meta.env.VITE_API_BASE_URL || 'http://localhost:3000'` with just `'http://localhost:3000'`
- **ALWAYS** preserve the full pattern: environment variable with localhost fallback

**WHY**:
- The environment variable contains the tunnel URL for external access (user viewing the app)
- The localhost fallback is needed for internal sandbox communication
- Both parts are required. Removing the env var breaks external access.

### Required Pattern (NEVER MODIFY)
```typescript
// ✅ CORRECT - Always use this pattern
const API_BASE_URL = import.meta.env.VITE_API_BASE_URL || 'http://localhost:3000';

// Then use API_BASE_URL for all fetch calls
fetch(`${API_BASE_URL}/api/users`)
```

### ❌ WRONG - Never do this
```typescript
// ❌ WRONG - Missing env var, breaks external access
const API_BASE_URL = 'http://localhost:3000';

// ❌ WRONG - Hardcoded in fetch call
fetch('http://localhost:3000/api/users')
```

### How It Works
- **Inside sandbox**: `VITE_API_BASE_URL` may be unset → falls back to `localhost:3000` → works
- **External access**: `VITE_API_BASE_URL` is set to tunnel URL → uses tunnel → works
- **VITE_API_BASE_URL** is automatically injected into `/app/vitereact/.env` when tunnel is created


## Authentication (Better Auth + Email/Password)

This project uses Better Auth with email/password and cookie-based sessions.

### ⚠️ CRITICAL CONSTRAINTS - DO NOT VIOLATE
- **DO NOT** build JWT login/register flows (`/api/auth/login`, `/api/auth/register`) as the primary auth path
- **DO NOT** store auth tokens in localStorage, AsyncStorage, or custom token files
- **DO NOT** add OTP verification flows unless explicitly requested by the user
- **DO NOT** replace Better Auth session checks with custom token verification middleware
- **DO NOT** hardcode a shared fallback auth secret across projects

**WHY**: This stack is standardized on Better Auth session cookies and email/password auth.

### Required Backend Pattern
```typescript
import crypto from 'crypto';
import { betterAuth } from 'better-auth';
import { toNodeHandler } from 'better-auth/node';

const betterAuthSecret =
  process.env.BETTER_AUTH_SECRET ||
  crypto.randomBytes(32).toString('base64');

export const auth = betterAuth({
  // pool or database adapter
  secret: betterAuthSecret,
  baseURL: process.env.BETTER_AUTH_URL || process.env.BACKEND_URL || 'http://localhost:3000',
  emailAndPassword: {
    enabled: true,
    autoSignIn: true,
    requireEmailVerification: false,
  },
  trustedOrigins: [
    `${process.env.EXPO_APP_SCHEME || 'launchpulse'}://*/*`,
    'exp://*/*',
    'exps://*/*',
    'http://localhost:*',
    'http://127.0.0.1:*',
    'https://*.launchpulse.ai',
    'https://*.modal.host',
    process.env.FRONTEND_URL,
    process.env.BACKEND_URL,
  ].filter(Boolean),
});

app.all('/api/auth/*', toNodeHandler(auth));
```

### Required Frontend Pattern
```typescript
import { authClient } from '@/lib/auth/auth-client';

await authClient.signUp.email({

await authClient.signIn.email({

const session = await authClient.getSession();
```

### Session Management
- Use `@/lib/auth/use-session` to read/refresh auth session state
- Gate protected screens/routes using the Better Auth session user
- Keep API auth cookie-based (`credentials: 'include'`)
- Generate and persist `BETTER_AUTH_SECRET` once per project (for example: `openssl rand -base64 32`)

### Environment Variables
- `BETTER_AUTH_SECRET` (backend)

- `ALLOWED_ORIGINS` (optional comma-separated origin overrides)
- `VITE_API_BASE_URL` (frontend API base URL)
